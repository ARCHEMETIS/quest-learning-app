# Design Brief: ออกแบบ UI ลุยเควส (LuiQuest) ใน Claude

> เอกสารนี้สำหรับ**เพื่อนผู้ออกแบบ UI** — อ่าน brief นี้ให้จบก่อน แล้ว copy พรอมต์ท้ายไฟล์ (section "พรอมต์พร้อมวาง") ไปวางใน Claude (claude.ai) เพื่อเริ่มออกแบบได้ทันที พรอมต์นั้น self-contained มีข้อมูลครบในตัว ไม่ต้องแนบไฟล์อื่น

---

## 1. แอพนี้คืออะไร

**ลุยเควส (LuiQuest)** — แอพเรียนรู้แบบเควสรายวัน ภาษาไทย สำหรับนักศึกษาไทยสายไอที/วิศวะ/วิทย์

- **Tagline:** "อยากเก่งอะไร ลุยเลย — วันละเควส"
- ผู้ใช้เลือกหัวข้อที่อยากเก่ง (เช่น Python, Excel, การเงิน) → แอพสร้าง roadmap + เควสรายวันให้ → ทำเควส เก็บ XP สะสม streak แข่ง leaderboard ชวนเพื่อนมาลุยด้วยกัน
- โตด้วยปากต่อปากในกลุ่มมหาลัย → หน้าจอที่เกี่ยวกับการแชร์/ชวนเพื่อนสำคัญมาก

## 2. ข้อกำหนดตายตัว (ต้องเคารพ ห้ามหลุด)

1. **แบรนด์:** ชื่อ "ลุยเควส" (โรมัน: LuiQuest), tagline ตามข้อ 1
2. **โทน:** สนุก เป็นกันเองแบบไทย ๆ — มุกได้ meme ได้ เหมาะกับการแชร์ในกลุ่มมหาลัย; มีกลไกเกม (quest/XP/streak) แต่**ไม่ต้องเล่นคำ RPG จ๋า** (ไม่ต้องมี "ท่านนักผจญภัย" อะไรแบบนั้น)
3. **ภาษา:** UI ทั้งหมดเป็น**ภาษาไทย** (ศัพท์เทคนิคทับศัพท์ได้ตามธรรมชาติ เช่น XP, streak)
4. **Responsive:** **mobile-first** เป็นหลัก + ต้องดูดีบน **iPad/tablet** (ไม่ใช่แค่ยืดกว้าง — จัด layout ให้เหมาะ)
5. **PWA:** แอพติดตั้งลงเครื่องได้ → ออกแบบให้เป็น standalone app (มี navigation ในตัว ไม่พึ่ง back button ของ browser)
6. **รูปแบบส่งงาน:** **React component (JSX + Tailwind CSS)** — เงื่อนไขใน section 5

## 3. รายการหน้าจอที่ต้องออกแบบ (10 หน้า + app shell)

### 3.1 Login
- แสดง: โลโก้/ชื่อแอพ + tagline + ปุ่ม **"เข้าสู่ระบบด้วย Google"** (เป็นทางเดียว — ไม่มี email/password, ไม่มี guest) + ประโยคขายสั้น ๆ ว่าแอพทำอะไร
- Edge states: กำลังล็อกอิน (loading), ล็อกอินล้มเหลว (error + ลองใหม่), **มาจากลิงก์ชวนเพื่อน** → มี banner "เพื่อนชวนคุณมาลุย" (แสดงชื่อคนชวนได้)

### 3.2–3.4 Onboarding 3 ขั้น (จบใน <1 นาที — มี progress indicator 1/2/3)
- **ขั้น 1 เลือกหัวข้อ:** การ์ด 6 หัวข้อ curated — Python, Data/ML, สร้างเว็บ, ใช้ AI เป็น, Excel, การเงินส่วนบุคคล — **+ ช่องพิมพ์หัวข้ออิสระ** (อยากเรียนอะไรก็พิมพ์เอง)
- **ขั้น 2 ระดับพื้นฐาน:** 3 ตัวเลือก — มือใหม่ / พอมีพื้น / แน่นแล้ว
- **ขั้น 3 เวลาว่างต่อวัน:** 3 ตัวเลือก — 15 / 30 / 60 นาที
- จบขั้น 3 → ไปหน้าเควสแรก**ทันที** (หัวข้อ curated มี starter quest สำเร็จรูป ขึ้น 0 วินาที)
- Edge state: **หัวข้อพิมพ์อิสระ** ต้อง generate สด → มี**หน้ารอ** (loading state สนุก ๆ ไม่น่าเบื่อ, ~5–20 วินาที)
- ชื่อ/รูปโปรไฟล์ดึงจาก Google อัตโนมัติ — **ไม่มีขั้นถามชื่อ**

### 3.5 หน้าเควสรายวัน (หน้าหลักหลังล็อกอิน — หน้าที่ผู้ใช้เห็นบ่อยสุด)
- แสดง: เควสของวันนี้ (ชื่อ + คำอธิบาย + ลิงก์แหล่งเรียน), **checklist ของเควส** (ติ๊กทีละข้อ — **ต้องติ๊กครบทุกข้อถึงได้ XP** = gating), แถบสรุปสถานะ: **XP สะสม, streak (จำนวนวันติด), phase progress ของ roadmap (คืบหน้ากี่ % / phase ไหน), letter grade (A/B/C…)**, ทางเข้าแชทโค้ช
- Edge states:
  - **โหลด** (skeleton — ปกติเร็วเพราะเควส pre-generate ไว้แล้ว)
  - **เควสวันนี้ยังไม่พร้อม** (ระบบ generate ไม่ทัน/ล้ม → ข้อความอธิบาย + ปุ่มลองใหม่ ไม่ใช่จอขาว)
  - **ทำเควสวันนี้เสร็จแล้ว** (celebration + XP ที่ได้ + CTA ชวนแชร์การ์ด streak / ชวนเพื่อน)
  - **streak ขาด** (พลาดไปวัน) — โทนปลอบ+ยุให้กลับมาลุยใหม่ ไม่ประณามผู้ใช้
  - **ผู้ใช้ใหม่วันแรก** (ยังไม่มีสถิติ — ตัวเลขเป็น 0 ต้องยังดูมีชีวิตชีวา ไม่โล่งหลอน)

### 3.6 แชทโค้ช AI
- แสดง: บทสนทนา (bubble ผู้ใช้/โค้ช), ช่องพิมพ์, **ตัวนับโควต้า "เหลือ X/10 ข้อความวันนี้"** เห็นชัดตลอด
- Edge states:
  - **ว่าง** (ยังไม่เคยคุย → ข้อความต้อนรับ + ปุ่มคำถามแนะนำ 2–3 ปุ่ม)
  - **โค้ชกำลังพิมพ์** (typing indicator)
  - **โควต้าหมด (0/10)** — ช่องพิมพ์ disabled + บอกชัดว่ารีเซ็ตเมื่อไหร่ (เที่ยงคืน) + ชวนไปทำอย่างอื่นต่อ (ทำเควส/อ่านแหล่งเรียน) — โทนไม่หงุดหงิดใส่ผู้ใช้
  - **ส่งไม่สำเร็จ** (ระบบล่ม/เน็ตหลุด → retry)

### 3.7 Leaderboard
- แสดง: อันดับ XP รวมทั้งแอพ — รูป+ชื่อ (จาก Google), XP, อันดับ; **แถวของตัวเองต้องหาเจอง่าย** (highlight/pin ไว้แม้อันดับลึก)
- Edge states: โหลด, **คนน้อยช่วงแอพเพิ่งเปิด** (เช่น มี 5 คน → ต้องไม่ดูโล่งเหงา + CTA ชวนเพื่อนมาเติมอันดับ)

### 3.8 ชวนเพื่อน (invite)
- แสดง: ลิงก์ชวนส่วนตัว + ปุ่ม copy + ปุ่มแชร์ (LINE / IG / อื่น ๆ), คำอธิบายรางวัล (**สมัครผ่านลิงก์ = ได้ XP ทั้งคู่**), ยอดเพื่อนที่สมัครผ่านลิงก์เราแล้ว
- Edge states: copy สำเร็จ (feedback), **ยังไม่มีใครสมัครผ่านลิงก์** (ว่าง — ยุให้เริ่มแชร์)

### 3.9 การ์ดแชร์ streak
- แสดง: การ์ดภาพสำหรับโพสต์ **IG story (แนวตั้ง 9:16)** เป็นหลัก — มี streak เด่น ๆ, ชื่อผู้ใช้, แบรนด์ลุยเควส + tagline/ลิงก์ชวน (การ์ดคือป้ายโฆษณาเคลื่อนที่ของแอพ)
- หน้าจอรอบการ์ด: preview + ปุ่มแชร์/บันทึกภาพ
- Edge state: **streak = 0 หรือเพิ่งขาด** → เปลี่ยนข้อความบนการ์ด (เช่น อวด XP รวมแทน) ไม่ใช่โชว์เลข 0 เฉย ๆ

### 3.10 หน้า /stats สาธารณะ (ยืนยันแล้ว — ต้องออกแบบ)
- **หน้าสาธารณะ ไม่ต้องล็อกอิน** (คนนอก/อาจารย์เปิดลิงก์ดูได้เลย) — ทำหน้าที่โชว์การเติบโตของแอพ + เป็น social proof ให้คนที่แชร์กันปากต่อปาก
- แสดง:
  1. **ตัวเลข headline 2 ตัว** — "สมัครแล้ว N คน" + "ใช้งานจริง M คน" (ตัว "ใช้งานจริง" เด่นกว่า)
  2. **กราฟการเติบโตสะสม** (เส้นโตขึ้นตามวัน) ของยอดผู้ใช้
  3. **ตัวเลขรอง social-proof** — จำนวนเควสที่ทำเสร็จรวมทั้งแอพ, streak สูงสุด/เฉลี่ย
  4. แบรนด์ลุยเควส + CTA ชวนสมัคร (หน้านี้คนนอกเห็น → ควรมีปุ่มไปหน้า login)
- **ห้ามมีข้อมูลส่วนตัว/ชื่อผู้ใช้** ในหน้านี้ (aggregate เท่านั้น — ต่างจาก leaderboard)
- Edge states: โหลด, **ช่วงเปิดตัวเลขยังน้อย** (ต้องดูมีอนาคต ไม่โล่งเหงา)

## 4. อาจมีเพิ่ม (รอยืนยัน — ยังไม่ต้องทำ)

- **ฟีเจอร์ Wave 2** (มาหลังเปิดตัว — ยังไม่ต้องออกแบบ แต่รู้ไว้เผื่อเว้นที่): web push แจ้งเตือนกัน streak ขาด, ระบบเพื่อน/duo (เห็น streak กันและกัน), ชั้นจ่ายเงิน premium (เรียนหลายหัวข้อพร้อมกัน — ฟรีได้ทีละ 1 หัวข้อ)

## 5. รูปแบบการส่งงาน

- **React function component — JSX + Tailwind CSS เท่านั้น** (ไม่ใช่ TypeScript, ไม่ใช้ CSS file แยก, ไม่ใช้ UI library ภายนอกอย่าง MUI/shadcn — เพราะฝั่งโค้ดจะ copy ลงโปรเจกต์ Vite+React+Tailwind ตรง ๆ)
- 1 หน้าจอ = 1 component หลัก, mock data ประกาศไว้บนหัวไฟล์ (เดี๋ยวฝั่งโค้ดต่อ API เอง)
- ทุก edge state ที่ระบุในข้อ 3 ต้องมีให้เห็นในงาน (จะทำเป็น state toggle ใน component หรือแยก variant ก็ได้)
- icon ใช้ SVG inline หรือ emoji ได้ ไม่ต้องพึ่ง icon library

## 6. สิ่งที่เปิดกว้าง 100% — เพื่อนตัดสินใจเองทั้งหมด

- **โทนสี / palette** (light/dark ก็แล้วแต่)
- **Layout** ทุกหน้า + รูปแบบ navigation
- **Typography** (แนะนำ font ไทยที่อ่านง่ายบนจอ เช่นตระกูล Google Fonts ไทย — แต่เลือกเองเลย)
- **ธีมความเป็นเกม** — จะเกมจัด จะ minimal มี XP แอบ ๆ ก็ได้ ขอแค่โทนแบรนด์ตามข้อ 2
- **ภาพประกอบ / mascot / animation** — มีหรือไม่มีก็ได้

ดีไซน์เก่าของแอพรุ่นก่อน (ML Quest) **ทิ้งไปแล้ว** — ไม่ต้องดู ไม่ต้องอิง เริ่มจากศูนย์ได้เลย

## 7. วิธีทำงานใน Claude (แนะนำ)

1. วางพรอมต์ข้างล่างใน Claude → Claude จะเสนอ design direction (สี/ฟอนต์/ธีม) มาให้เลือกก่อน
2. เคาะ direction แล้วให้ Claude ทำทีละหน้า เป็น artifact (React) — เห็น preview สด ๆ ใน Claude เลย
3. ทวนแต่ละหน้าว่า edge states ครบตาม brief ไหม แล้วสั่งแก้จนพอใจ
4. เสร็จแล้ว copy โค้ด JSX ของแต่ละหน้าส่งกลับมา (ไฟล์ .jsx หรือวางในแชทก็ได้)

   states: โหลด (skeleton) / เควสยังไม่พร้อม (ข้อความ+ปุ่มลองใหม่) / ทำเสร็จแล้ววันนี้ (celebration + ชวนแชร์การ์ด streak) / streak ขาด (โทนปลอบ+ยุให้กลับมาลุย ไม่ประณาม) / ผู้ใช้ใหม่วันแรก (เลขเป็น 0 แต่ต้องไม่โล่งหลอน)

6) แชทโค้ช AI — bubble สนทนา, ช่องพิมพ์, ตัวนับโควต้า "เหลือ X/10 ข้อความวันนี้" เห็นชัดตลอด
   states: ว่าง (ข้อความต้อนรับ + ปุ่มคำถามแนะนำ 2–3 ปุ่ม) / โค้ชกำลังพิมพ์ / โควต้าหมด (ช่องพิมพ์ disabled + บอกว่ารีเซ็ตเที่ยงคืน + ชวนไปทำเควสต่อ โทนไม่กวนใจ) / ส่งไม่สำเร็จ (retry)

7) Leaderboard — อันดับ XP รวมทั้งแอพ (รูป+ชื่อจาก Google, XP, อันดับ), แถวตัวเอง highlight/pin ให้หาเจอแม้อันดับลึก
   states: โหลด / คนน้อยช่วงเปิดตัว (มีแค่ ~5 คน ต้องไม่ดูเหงา + CTA ชวนเพื่อนมาเติมอันดับ)

8) ชวนเพื่อน — ลิงก์ชวนส่วนตัว + ปุ่ม copy + ปุ่มแชร์ (LINE/IG/อื่น ๆ), อธิบายรางวัล "สมัครผ่านลิงก์ = ได้ XP ทั้งคู่", ยอดเพื่อนที่สมัครแล้ว
   states: copy สำเร็จ / ยังไม่มีใครสมัคร (ยุให้เริ่มแชร์)

9) การ์ดแชร์ streak — การ์ดภาพแนวตั้ง 9:16 สำหรับ IG story: streak เด่น ๆ, ชื่อผู้ใช้, แบรนด์ลุยเควส + tagline (การ์ดคือป้ายโฆษณาของแอพ) + หน้าจอ preview พร้อมปุ่มแชร์/บันทึกภาพ
   state: streak = 0 หรือเพิ่งขาด → การ์ดเปลี่ยนไปอวด XP รวมแทน ไม่โชว์เลข 0

10) หน้า /stats สาธารณะ (ไม่ต้องล็อกอิน คนนอก/อาจารย์เปิดดูได้) — โชว์การเติบโตของแอพเป็น social proof: ตัวเลข headline 2 ตัว "สมัครแล้ว N คน" + "ใช้งานจริง M คน" (ตัวใช้งานจริงเด่นกว่า), กราฟการเติบโตสะสมของยอดผู้ใช้ตามวัน, ตัวเลขรอง (จำนวนเควสที่ทำเสร็จรวม, streak สูงสุด/เฉลี่ย), แบรนด์ + ปุ่มชวนสมัคร (ไป login) — ห้ามมีชื่อ/ข้อมูลส่วนตัว aggregate เท่านั้น
   states: โหลด / ช่วงเปิดตัวเลขยังน้อย (ต้องดูมีอนาคต ไม่โล่งเหงา)

11) App shell / navigation — ทางไปหน้า: เควสวันนี้, แชทโค้ช, leaderboard, ชวนเพื่อน + ที่แสดงรูป/ชื่อผู้ใช้ (ไม่ต้องมีหน้า profile แยก) — รูปแบบ nav คุณเลือกเอง (/stats เป็นหน้าสาธารณะนอก nav หลัก ไม่ต้องใส่ในแถบ nav ของผู้ใช้ที่ล็อกอิน)

หมายเหตุ: มีฟีเจอร์เฟสถัดไปที่ยังไม่ต้องออกแบบ (push notification, ระบบเพื่อน/duo, ชั้นจ่ายเงิน premium เรียนหลายหัวข้อ) — รู้ไว้เฉย ๆ เผื่อออกแบบ nav ให้ขยายได้
