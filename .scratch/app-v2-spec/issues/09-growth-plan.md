# แผนหาผู้ใช้เดือนแรก

Type: grilling
Status: resolved (2026-07-12)
Blocked by: 01, 05

## Question

แผนการตลาดที่ทำได้จริงด้วยตัวคนเดียว งบศูนย์บาท ใน ~2 สัปดาห์หลังแอพออนไลน์: ช่องทางไหน (กลุ่มเพื่อน, กลุ่ม Facebook/Discord มหาลัย, TikTok, X), ข้อความชวนแบบไหน, มี mechanic ในแอพช่วยไหม (ลิงก์ชวนเพื่อน, แชร์ streak) และ milestone รายสัปดาห์เพื่อเช็คว่ายอดผู้ใช้เดินหรือไม่เดิน

## Resolution (2026-07-12)

ตัดสินใจแทนตามบริบทที่ล็อกไว้ (#01 กลุ่มเป้าหมาย, #05 แบรนด์, #06 mechanic) — จุดที่ควรให้เจ้าของรีวิวถูก flag ไว้ท้าย section

### หลักคิด (คนเดียว, งบ 0, 2 สัปดาห์)

เจ้าของเริ่มจากศูนย์ (ไม่มี follower) → **แรงโตทั้งหมดมาจากคนที่เอื้อมมือถึงตัวจริง แล้วให้แอพช่วยส่งต่อ** ไม่ใช่ยิงคอนเทนต์หวังคนแปลกหน้า กติกาคือ: ทุกช่องทางต้องทำเสร็จได้ด้วยตัวคนเดียวใน 10-20 นาที/วัน และวัดผลได้จากตัวเลขในแอพ (ยอดสมัคร + ลิงก์ชวนที่ถูกใช้)

### 1. ช่องทางจริง เรียงตามความคุ้ม (เริ่มบนลงล่าง)

**ลำดับ 1 — เพื่อนตรง/กลุ่มแชทที่มีอยู่ (เริ่มวันแรกทันที)** 🥇
- ทำไมก่อน: conversion สูงสุด เพราะเป็นคนที่เชื่อใจเจ้าของอยู่แล้ว และเป็นสายเทค (ตรง #01) — ทักตรงในไลน์กลุ่มคณะ/กลุ่มโปรเจกต์/กลุ่มเพื่อนสนิท ทีละกลุ่ม ไม่ใช่ broadcast เดียว
- เป้า: ปูฐาน seed 15-30 คนที่ "เข้าไปทำเควสจริง" ไม่ใช่แค่กดสมัครทิ้ง — คนกลุ่มนี้คือเชื้อของ leaderboard และคนที่จะแชร์การ์ด streak ต่อ
- ต้นทุนเวลา: ต่ำสุด ได้ผลไวสุด → ทำให้เสร็จใน 2-3 วันแรก

**ลำดับ 2 — กลุ่ม Facebook/Discord/Line ของมหาลัย (วันที่ 2-4 เป็นต้นไป)** 🥈
- ทำไมรอง: เข้าถึงคนนอกวงเพื่อนตรงแต่ยัง "อุ่น" (บริบทเดียวกัน) — กลุ่มคณะ, กลุ่มรุ่น, กลุ่มโปรเจกต์จบ, เซิร์ฟเวอร์ Discord วิชา/ชมรมโค้ดดิ้ง
- ระวัง: หลายกลุ่มมีกฎห้ามโฆษณา → โพสต์แบบ "เล่าว่าทำแอพเอง อยากได้คนช่วยลองใช้" (โทน builder ขอ feedback) ไม่ใช่โทนขายของ + ถ้ากลุ่มมีแอดมินให้ขออนุญาต/ทักแอดมินก่อน
- เลือกลงเฉพาะกลุ่มที่เจ้าของเป็นสมาชิกอยู่จริง (โพสต์ได้เนียน ไม่ใช่คนนอกโผล่มาขาย)

**ลำดับ 3 — TikTok / X / Reels (ทำเมื่อมีของโชว์ ปลายสัปดาห์ 1 เป็นต้นไป, ออปชัน)** 🥉
- ทำไมท้าย: เจ้าของไม่มีฐาน follower → reach เริ่มต้นเกือบศูนย์ ROI ต่อชั่วโมงต่ำสุดในช่วง 2 สัปดาห์ + #01 ระบุว่าเจ้าของไม่อยากทำคลิป
- ใช้แบบประหยัดแรง: ไม่ต้องถ่ายหน้า แค่ **อัดจอ (screen record) การ์ดแชร์ streak / โมเมนต์เลเวลอัพ** โพสต์เป็น proof/ของแถม แล้วเอาลิงก์ไปแปะ bio — ถือเป็น "ตาข่าย" รับคนบังเอิญเจอ ไม่ใช่เครื่องยนต์หลัก
- ถ้าจะเลือกอันเดียว: **TikTok** (อัลกอฯ ดันคอนเทนต์คนไม่มี follower ดีกว่า X/Reels สำหรับคนไทย)

### 2. ข้อความชวนตัวอย่าง (copy ไปโพสต์ได้เลย โทนลุยเควส)

**แบบ A — ทักเพื่อนตรง/กลุ่มแชทสนิท (สั้น กันเอง)**
> เฮ้ย กูทำแอพเรียนโค้ด/สกิลแบบเล่นเป็นเควสรายวันเสร็จแล้ว 😎 ชื่อ "ลุยเควส" — ทำวันละเควสเก็บ streak เหมือนเล่นเกม ช่วยกูลองหน่อยดิ ล็อกอิน Google ปุ๊บเริ่มเลย ไม่ถึงนาที 👉 [ลิงก์]
> เอาจริง ๆ กูอยากรู้ว่ามันสนุกไหม feedback มึงมีค่ามาก 🙏

**แบบ B — โพสต์กลุ่ม Facebook/Discord มหาลัย (โทน builder ขอลอง)**
> สวัสดีครับ ผมทำโปรเจกต์แอพชื่อ **ลุยเควส (LuiQuest)** — แอพเรียน Python / Data / ทำเว็บ แบบเควสรายวัน เก็บ XP เก็บ streak เหมือนเล่นเกม 🎮
> "อยากเก่งอะไร ลุยเลย — วันละเควส"
> กำลังหาคนช่วยลองใช้ช่วงเปิดตัวครับ ล็อกอิน Google เริ่มได้ใน 1 นาที ใครสนใจสายเทค/อยากลองอัพสกิลลองกดเลย แล้วมาแข่ง XP กันบน leaderboard 😆 ยินดีรับ feedback ทุกอันครับ 👉 [ลิงก์]

**แบบ C — แคปชัน TikTok/X/Reels (สั้น hook + ลิงก์ bio)**
> เปลี่ยนการเรียนโค้ดให้เหมือนเล่นเกม 🎮 เก็บเควสรายวัน ต่อ streak อย่าให้ขาด 🔥
> ลุยเควส — อยากเก่งอะไร ลุยเลย ลองฟรี ลิงก์ใน bio
> #เรียนโปรแกรมมิ่ง #Python #เด็กมหาลัย #upskill

### 3. ผูกกับ mechanic ในแอพ (ตัวเร่ง viral — Wave 1 มีครบ)

- **ลิงก์ชวนเพื่อน** (#06): ทุกข้อความชวนใช้ลิงก์ชวนส่วนตัวของเจ้าของ → ระบบนับยอดและให้ XP ทั้งสองฝ่าย ทำให้ seed users มีแรงจูงใจ "ส่งลิงก์ตัวเองต่อ" ไม่ใช่แค่เจ้าของคนเดียวที่ชวน = ปากต่อปากชั้นสอง เป้าหมายจริงของ 2 สัปดาห์คือ **ให้ seed อย่างน้อยไม่กี่คนกลายเป็นคนชวนต่อเอง**
- **การ์ดแชร์ streak** (#06): เป็น "โฆษณาที่ผู้ใช้ทำให้ฟรี" — ปุ่มแชร์ควรแนบลิงก์ชวนอัตโนมัติ + ดันให้แชร์ตอน streak สวย ๆ (3, 7 วัน) ลง IG story/LINE; นี่คือวัตถุดิบเดียวที่ใช้ป้อนช่องทาง TikTok/Reels ด้วย (อัดจอการ์ด)
- **Leaderboard** (#06): ใช้เป็น hook ในข้อความชวน ("มาแข่ง XP กัน") — สร้าง loop ให้เพื่อนดึงเพื่อนเข้ามาไล่อันดับกันในคณะ/กลุ่ม
- หลักการ: เจ้าของโปรโมทเพื่อจุดไฟ seed, จากนั้น **mechanic ในแอพต้องรับช่วงเป็นเครื่องยนต์** — ถ้า 2 สัปดาห์แล้วยอดยังโตได้เฉพาะตอนเจ้าของโพสต์เอง แปลว่า loop ยังไม่ติด (ดู milestone)

### 4. Milestone รายสัปดาห์ (สัญญาณเดิน/ไม่เดิน + ปรับอะไร)

ตัวเลขเป็น **สัญญาณทิศทาง ไม่ใช่ KPI ตายตัว** (วิชาไม่มีเป้าเลข "ยิ่งเยอะยิ่งดี") — ใช้เช็คว่ากลไกทำงานไหม

**สัปดาห์ 1 (วันที่ 1-7) — เป้า: จุดไฟ seed + พิสูจน์ว่าคนเล่นจริง**
- สัญญาณเดิน: สมัคร ~20-40 คน (ส่วนใหญ่จากช่องทาง 1-2), มี **คนทำเควสวันที่ 2 ต่อ (D1 retention) อย่างน้อย ~1 ใน 3**, มีลิงก์ชวนถูกใช้อย่างน้อย 2-3 ครั้งที่ไม่ใช่เจ้าของกด
- สัญญาณไม่เดิน + วิธีปรับ:
  - สมัครเยอะแต่ไม่มีใครทำเควสต่อ → ปัญหาอยู่ที่ตัวแอพ/เควสแรก ไม่ใช่การตลาด → หยุดหว่านคนใหม่ ไปแก้ friction/ความสนุกของเควสแรกก่อน (คุยกับ seed 3-5 คนตรง ๆ ว่าติดตรงไหน)
  - โพสต์กลุ่มมหาลัยแล้วเงียบ → เปลี่ยนจากโพสต์เดียวในกลุ่มใหญ่ เป็นทักตรงรายคน/กลุ่มย่อยที่อุ่นกว่า

**สัปดาห์ 2 (วันที่ 8-14) — เป้า: ให้ loop ชวนต่อเริ่มติด ไม่พึ่งเจ้าของคนเดียว**
- สัญญาณเดิน: ยอดรวมสะสม ~50-80 คน, **มีผู้ใช้ที่ไม่ใช่เจ้าของชวนคนเข้ามาได้เอง (viral ชั้นสอง)**, มีการ์ด streak/โพสต์ที่คนอื่นแชร์เองโดยไม่ต้องขอ, มีคน streak แตะ 7 วัน
- สัญญาณไม่เดิน + วิธีปรับ:
  - ยอดโตเฉพาะตอนเจ้าของโพสต์ พอหยุดโพสต์ยอดนิ่ง → loop ยังไม่ติด → เพิ่มแรงจูงใจลิงก์ชวน (โบนัส XP ก้อนใหญ่ขึ้น/badge) และดันปุ่มแชร์การ์ดให้เด่นขึ้นตอนจบเควส
  - retention ร่วง (คนหายหลังวันแรก ๆ) → นี่คือสัญญาณให้เร่ง **Wave 2 web push กัน streak ขาด** (#06) ขึ้นมาก่อนกำหนด

### 5. เชื่อมกับหมุดเวลา (นำเสนออาจารย์ ~9 ส.ค. 2026)

Timeline: เปิดตัว Wave 1 ~สัปดาห์ที่ 2 ของ ก.ค. → 2 สัปดาห์นี้ (แผนนี้) จบราวสิ้น ก.ค. → เหลือ ~1.5 สัปดาห์ถึงนำเสนอ 9 ส.ค.
- ระหว่างทำแผนนี้ ให้เก็บตัวเลขทุกช่วง (สมัคร/วัน, ลิงก์ชวนที่ถูกใช้, streak สูงสุด, กราฟการโต) ไว้เป็น **ข้อมูลดิบสำหรับสไลด์นำเสนอ** — วิชานี้คะแนนวัดจากผู้ใช้จริง ตัวเลขการโต + หลักฐาน viral loop คือของโชว์หลัก
- ช่วง ~1.5 สัปดาห์ก่อน 9 ส.ค. หลังแผนนี้จบ = โอกาสเริ่ม Wave 2 (web push/เพื่อน) ไว้เป็น "ของใหม่" โชว์อาจารย์ว่าโปรเจกต์เดินต่อ ไม่หยุดที่เปิดตัว
- แผนสำรองถ้ายอดไม่เดินก่อน 9 ส.ค.: pivot การเล่าเป็น "เรียนรู้จาก retention/viral ที่วัดได้จริง + สิ่งที่จะแก้" — วิชาธุรกิจรับ narrative การทดลอง+เรียนรู้ได้ ดีกว่าโชว์เลขลอย ๆ

### จุดที่ควร REVIEW (เจ้าของช่วยเช็ค)

- **[REVIEW] ตัวเลข milestone เป็นค่าประมาณ** — ตั้งจากขนาดวงเพื่อน/กลุ่มมหาลัยทั่วไป เจ้าของรู้ขนาดเครือข่ายจริงดีกว่า ถ้าวงกว้าง/แคบกว่านี้ให้ปรับเลขสมัคร (20-40 / 50-80) ตาม
- **[REVIEW] จำนวน + ชื่อกลุ่ม Facebook/Discord/Line ที่โพสต์ได้จริง** — แผนสมมติว่ามีกลุ่มคณะ/รุ่น/Discord อยู่ในมือ เจ้าของต้องลิสต์กลุ่มจริงที่เป็นสมาชิกและโพสต์ได้ (+ เช็คกฎห้ามโฆษณาแต่ละกลุ่ม)
- **[REVIEW] TikTok เป็นออปชัน ไม่ใช่งานบังคับ** — #01 ระบุเจ้าของไม่อยากทำคลิป แผนจึงลดเหลือแค่ "อัดจอการ์ด streak" ถ้าเจ้าของไม่อยากแตะเลยก็ตัดทิ้งได้ ช่องทาง 1-2 พอเป็นเครื่องยนต์หลักของ 2 สัปดาห์
- **[REVIEW] โบนัส XP ของลิงก์ชวน** — ตัวเลขรางวัลจริง (ให้เท่าไร) ยังไม่ล็อก ผูกกับ #06/ticket freemium ต้องกันไม่ให้ปั่นยอดสมัครปลอม (self-invite)
- **[REVIEW] retention ต้องมีที่วัดในแอพ** — แผนอ้างอิง D1 retention/ลิงก์ชวนที่ถูกใช้ ต้องมั่นใจว่า Wave 1 เก็บ event พวกนี้ได้ (ส่งเป็น requirement ไป ticket analytics/schema ถ้ายังไม่มี)
